# Qwen3-VL-8B-Thinking Text SFT for SWE-bench Multilingual

## 2026-06-08 Wrap-up Handoff

- No score-improving Qwen3-VL-8B-Thinking-text SFT checkpoint has been confirmed. The strongest clean reference remains the base checkpoint result in `eval/benchmark_results.md`: 2/30 on the predictive SWE-bench Multilingual subset.
- The local eval path was rechecked on the two historical base-smoke pass tasks, `gin-gonic__gin-3741` and `nlohmann__json-4237`. Scoring the saved prior passing patches through the current local SWE-bench scorer produced 2/2, so the scorer/container path is valid. A fresh base generation rerun first produced 0/2, then four more stochastic reruns produced 1/2, 0/2 with empty patches, 1/2, and 2/2. The base model has high generation variance on this tiny smoke, so do not judge future checkpoints from a single 2-task sample. Summary artifact: `/wbl-fast/usrs/ee/code-swe-data/deepswe-data-gen/runs/swebench_ml/qwen3-vl-8b-base-variance-20260608T1611Z-summary.json`.
- Training/runtime changes preserved in this wrap-up: the online packer now honors assistant messages marked `trainable: false` or `loss: false`, the Qwen3 SFT launch script supports `MIN_LR`, checkpoint restore, and configurable checkpoint save/consolidation format, and helper scripts/configs exist for Qwen3-VL text-checkpoint merging, raw JSONL resharding, SWE-bench diagnostic subset selection, and the v0-v55 dataset-building/eval sequence.
- Throughput-wise, the fastest stable Qwen3-VL SFT shape during this work was still 65,536-token packs with 1,048,576 global tokens/update, online tokenization/packing, and compile enabled. Later patch-recovery continuations commonly ran around 71k-75k TPS after warmup.
- Recommended next step: start from the best parser-stable branch rather than the latest loss-improved branch. v17-style first-turn/anti-repeat behavior remained the most stable foundation; patch-recovery-only continuations repeatedly failed to move the model from inspect/read loops into edit/diff/submit behavior under actual mini-swe-agent prompts. Gate any new checkpoint on multiple stochastic reruns of the 2-task smoke plus an easy 4-task smoke, and require non-empty source edits before spending a 30-task run.
